Mobile Game Automation & Reverse Engineering

A personal deep-dive into automation and memory-level reverse engineering of a real-time AR ARPG mobile game, Monster Hunter NOW (from Pokemon Go company). This project explored mobile system behavior, game memory, and scripting logic through practical experimentation.

  • OpenCV + Vysor: Used computer vision to automate monster hunting by detecting game states and simulating user input
  • Python Scripting: Designed real-time control flow to manage hunt timing, item usage, and in-battle decisions
  • Rooted Android: Gained elevated permissions to implement GPS spoofing for map-based gameplay
  • GameGuardian: Performed address-based memory inspection to modify weapon attack values during runtime
  • APK reverse engineering: Decompiled the game client to retrieve hidden assets and internal keys (for educational exploration)

The automation system could consistently hunt all monsters in the game without human intervention. As a result, I became one of the earliest players to reach max level and obtain a complete weapon collection, all through scripting and reverse engineering.
